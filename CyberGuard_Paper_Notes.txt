1. INTRODUCTION / ABSTRACT
CyberGuard is an AI-powered web application designed to help YouTube content 
creators and viewers detect, visualize, and autonomously moderate toxic comments 
underneath videos. By synergizing the YouTube Data API v3 with a highly optimized 
Natural Language Processing (NLP) model (ToxicBERT), CyberGuard provides a local 
analytics dashboard that breaks down cyberbullying into six distinct classifications, 
allowing active automated or manual moderation.

3. SYSTEM ARCHITECTURE & TECH STACK
- Backend Framework: Python / Flask
- Database (Persistence): SQLite (cyberguard.db)
- Machine Learning Layer: PyTorch & HuggingFace Transformers (CPU-optimized)
- Frontend: Vanilla HTML5/CSS3 (Dark Theme, Glassmorphism UI) with Vanilla JS
- Data Visualization: Chart.js (Radar/Bar/Doughnut charts)
- Report Generation: html2pdf.js (Client-side PDF generation)
- API Integrations: Google Cloud (YouTube Data API v3) & OAuth 2.0

4. KEY FEATURES & FUNCTIONALITY
A) Core Analysis Pipeline
   - Users input a YouTube video URL.
   - The backend uses the YouTube Data API to fetch up to 2,000 top-level comments.
   - Each comment undergoes inference via the ToxicBERT model.
   - Data is aggregated using Pandas to calculate global toxicity percent and category averages.

B) Comprehensive Dashboard
   - Visualizes Toxicity Ratio (Toxic vs. Clean) and Category Distribution.
   - Highlights the highest scoring toxic comments and top threats.
   - Displays an interactive top-50 comment data table.

C) Active YouTube Moderation (OAuth 2.0)
   - Video creators can click "Sign In with YouTube" to authenticate via OAuth 2.0 
     (using `https://www.googleapis.com/auth/youtube.force-ssl` scope).
   - Authenticated creators can natively DELETE or HIDE (reject) comments dynamically 
     from the dashboard via a secure `/api/moderate` backend endpoint.

D) Export & History
   - Generates client-side, printer-friendly PDF Reports with custom light-theme styling.
   - Offers raw tabular data downloads as CSV files.
   - Saves historical analysis runs in SQLite, allowing users to revisit older 
     dashboards instantly without re-running CPU-heavy inference models.

6. FUTURE SCOPE & LIMITATIONS
- Inference Speed: Operating on CPUs limits concurrent analysis speed; future iterations 
  could utilize cloud GPU endpoints.
- API Quotas: Relies on the standard 10,000 unit/day Google API quota limitation.
- Multi-Platform: Currently restricted to YouTube, but the pipeline is abstract 
  enough to support Reddit, TikTok, or Instagram endpoints in the future.
7. TECHNICAL CHALLENGES & SOLUTIONS
- API Pagination & Latency: Retrieving up to 2,000 comments from the YouTube Data API presented latency challenges. The solution involved utilizing paginated requests (`pageToken`) combined with robust try/except error handling customized for common YouTube errors (quotaExceeded, commentsDisabled).
- Zero-State & Inference Loading: Waiting for CPU-bound HuggingFace model inference to process hundreds of comments locally can take upwards of 10-20 seconds. This UX challenge was mitigated by implementing a dynamic JavaScript-based loading state that rotates contextual reassurements ("Running AI analysis...", "Building your dashboard...") to retain user engagement.
- Stateless Security: Integrating OAuth 2.0 without relying on a heavy backend authentication framework (like Flask-Login) required precise manipulation of Flask's signed HTTP-only cookies (`session`). Storing OAuth credential objects securely in the session dictionary allowed stateless but fully secured YouTube API calls on behalf of the creator.

8. UI/UX & DESIGN DECISIONS
- Visual Language: The application employs a "Glassmorphism" design system over a dark-themed canvas (`#0f172a`). By utilizing semi-transparent, backdrop-blurred containers, the UI creates depth and a premium "cyber-security" aesthetic.
- Color Psychology: The palette uses strict semantic accents for immediate cognitive recognition. Toxic/Danger elements use red (`#ef4444`), warnings use amber (`#f59e0b`), and safe elements use emerald (`#10b981`).
- Printer-Friendly Export: A specifically targeted "Light Mode" CSS override (`.pdf-exporting`) is injected dynamically during PDF Export (`html2pdf.js`). This guarantees that downloaded PDF reports are printer-friendly, highly legible on white paper, and strip away unnecessary UI navigation elements without permanently disrupting the user's sleek dark-mode web experience.

9. ETHICS & PRIVACY
Analyzing user-generated text for toxicity requires stringent ethical and privacy considerations:
- Data Minimization: CyberGuard operates statelessly concerning personally identifiable information (PII). No usernames, emails, or user accounts are stored in the SQLite database; the system purely archives the mathematical outcomes (analysis history) and the raw strings of the analyzed comments.
- Creator Control: By implementing `setModerationStatus('rejected')` as the primary "Hide" action, the app empowers creators with the final say, pushing comments into the "Held for Review" tab on YouTube rather than imposing immediate, irreversible censorship.
- Algorithmic Bias Awareness: The ToxicBERT model inherently carries biases derived from its original Jigsaw training dataset. To counter blind AI automation, the dashboard presents the exact probabilistic scores (e.g., 0.85 Danger) alongside the text, emphasizing that the AI acts as an analytical assist tool while the human creator makes the final moderation decision.
